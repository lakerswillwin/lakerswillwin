hw3 b+tree and heap

  • 补充知识：

    1. b+tree:T(M,N)(insert)=O((M/log M)log N);Depth(M,N)=$O(log_{[M/2]}N)$;$T_{Find}(M,N)=O(log N)$
    2. NPl:The null path length.Npl(X) = min { Npl(C) + 1 for all C as children of X }

  • 知识1 b+树的定义与性质

    1. 根是叶子或者有2到M个孩子
    2. 所有非叶子的节点（除了根以外），有[M/2]到M个孩子，其本身最多存M-1个值
    3. 所有的叶子的深度是一样的（真实数据都存储在叶子中）
    4. 在空间最浪费的情况下是一颗[M/2]叉树，the deep of it is $O(log_{[M/2]}N)$

  • 知识2 b+树的操作

    1. 查找：类似于不断比较，根据非叶节点的信息找到对应的叶子，然后从叶子中查找。
    2. 插入：首先用与查找相同的方式找到应该插入的位置，然后插入。但是有可能导致叶子节点的数量超过M，此时需要分裂。而分裂会导致家长节点的家长节点的子节点变多，矛盾可能上移。

    3. 删除：首先和查找一样找到待删除的点。直接删除后可能不会直接导致下溢的产生。若可以向兄弟节点(通常先向右兄弟借键)借键，如果兄弟都不足以把键借出，则进行合并操作，并更新父节点。然后就进行非叶节点即索引的判断，如果索引关键字数量不满足且有兄弟节点可借（父节点下移到这个索引中，兄弟节点上移），否则父节点下移合并成新的索引。

  • 知识3 左偏堆（lefttist heap）的定义

    1. dist in leftist heap：如果一个结点的左孩子或右孩子为空结点则其dist为零，称为外结点。否则$dist=min(dist_{left child},dist_{right child})+1$
    2. leftist heap:左偏堆是满足结点的左孩子的dist不小于右孩子的dist的堆（这个定义与合并时有关）

  • 知识4 左偏堆的操作

    1. 合并（核心操作）：先维护堆的性质，在回溯时维护左偏性质。（实际上是一个自上而下再自下而上的过程）具体操作，先比较当前两个待合并子树的根结点的键值，选择较小（较大）的那个作为根结点，其左子树依然为左子树，右子树更新为「右子树和另一个待合并子树的合并结果」。当然，在递归地更新完后，我们需要检查左子树和右子树是否满足$dist_{left child}>=dist_{right child}$,如果不满足则交换左右子树。
    2. 单点插入：本质上就是可以看作合并只有一个结点的左偏堆
    3. 单点删除：合并需要被删除的结点的两个子结点，将这个新的树的根代替被删除的结点，然后再回溯更新dist调整即可。$O(logn)$

  • 知识5 一个lamma
    For a leftist heap with n nodes,its right path from the root to null has most $log_{2}(n+1)$ nodes

二向队列(binomial queue)

  • 知识1 二向队列的定义与性质
    1. 本质上是一系列二项树的集合
    2. k阶二项树是两个k-1阶二项树合并（直接令其中一颗树成为另外一颗树的新的child）得到
  • 知识2 二向队列的操作
    1. 首先我们可以利用特征比特向量来理解二向队列的系列操作（1011就可以对应$B_{3}B_{1}B_{0}$）

  1. 描述：K 中心问题指：给定平面上的一系列site，在平面中找出k个不同的center，记$site_{i}$到离它最近的center的距离为$dis_{i}$，求$max{dis_{i}}$的min

    2.当还有点没被覆盖时，重复这些操作：
    a.随机选取一个site，$s_{i}\in S_{x}$,将其插入$C_{x}$,并在$S_{x}$删除$s_{i}$
    b.删除Sx中所有距离si不足rx的点

  8. 当我们明白了上面的lemma以后，我们就可以根据我们的算法得到一个最终的$r_{x0}$,满足$r_{x0}\leq r(C^{*}) \leq 2r_{x0}$,此时显然易见近似比为2

    a.选取一个site $s_{i} \in S_{x}$,它满足$\forall s_{j} \in S_{x},dis(s_{i},C_{x}) \geq dis(s_{j},C_{x})$（即这个点是还没被覆盖的点中距离$C_{x}$最远的点）
